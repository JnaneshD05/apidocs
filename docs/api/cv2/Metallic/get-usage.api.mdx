## Get Usage



This API can be used to obtain usage data with various search parameters. All dates in usage API refer to UTC time zone. Usage data on each usage date shows the peak usage up to that usage date in current calendar month. Partner can either directly call API with no parameters to get usage data for yesterday (which is current calendar month peak usage up to yesterday) or specify date parameters to get historical peak usage data. Usage collection runs at 4am UTC every day for yesterday's usage data. Partner is suggested to call usage api for yesterday's data after 6am UTC.

##### **Billing Unit Explained**

| **Billing Unit** | **Description** | **Example Given** | **Example Calculation** |
| --- | --- | --- | --- |
| "billingUnit" is "1 user" | In this case we charge based on number of users that use our service. "consumedQuantity" is the peak count of users up to the usageDate in current calendar month. "chargeableQuantity" will be the same as "consumedQuantity", and unitPrice is what we charge per user. | consumedQuantity = 100  chargeableQuantity = 100  billingUnit = 1 user  unitPrice = 3 | billingQuantity = Ceiling of (chargeableQuantity / billingUnit) = Ceiling (100 users / 1 user) = 100  totalCost = billingQuantity \* unitPrice = 100 \* 3 = 300 |
| "billingUnit" is "1 tb" | In this case we charge based on number of terabytes that are used. "consumedQuantity" is the peak count of terabytes up to the usageDate in current calendar month. "chargeableQuantity" will be the same as "consumedQuantity", and unitPrice is what we charge per terabyte. | consumedQuantity = 100  chargeableQuantity = 100  billingUnit = 10 tb  unitPrice = 50 | billingQuantity = Ceiling of (chargeableQuantity / billingUnit) = Ceiling (100 tb / 10 tb) = 10  totalCost = billingQuantity \* unitPrice = 10 \* 50 = 500 |
| "billingUnit" is "10 vm" | In this case we charge based on number of VM bundles that are used. Each bundle contains 10 VM. "consumedQuantity" is the peak count of VM bundles up to the usageDate in current calendar month. "chargeableQuantity" could be different from "consumedQuantity". if customer has other services that run on VM, and unitPrice is what we charge per VM bundle. | consumedQuantity = 100  chargeableQuantity = 85  billingUnit = 10 vm  unitPrice = 5 | billingQuantity = Ceiling of (chargeableQuantity / billingUnit) = Ceiling (85 vm / 10 vm) = 9  totalCost = billingQuantity \* unitPrice = 9 \* 5 = 45 |
